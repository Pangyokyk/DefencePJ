## Project

Unity 6 game project (version 6000.3.18f1) using the Universal Render Pipeline (URP). Likely a tower defense game — AI Navigation (2.0.13) is installed. Target platform: Windows Standalone 64-bit.

## Unity-Specific Conventions

- **Never edit files in `Library/`, `Temp/`, or `Logs/`** — these are auto-generated by Unity and wiped on import/rebuild. All persistent code and assets live under `Assets/` or `Packages/`.
- **Script placement**: Runtime scripts go under `Assets/Scripts/` (create this directory). Editor-only scripts must be placed inside an `Editor/` folder to be excluded from runtime builds.
- **Assembly definitions**: The project uses implicit Assembly-CSharp (runtime) and Assembly-CSharp-Editor (editor-only) assemblies. Do not create `.asmdef` files unless explicitly requested.
- **Scene registration**: New scenes must be added to **File → Build Settings → Scenes In Build** to be loadable at runtime.
- **URP materials**: Use URP-compatible shaders (Lit, Unlit, etc.) — Built-in pipeline shaders will appear pink at runtime.

## Language and Style

- C# 9.0, targeting .NET Framework 4.7.1
- `AllowUnsafeBlocks` is disabled — do not use `unsafe` code
- Suppress warnings 0169 (unused fields) and USG0001 are already configured in the `.csproj`
- Use `[SerializeField]` for inspector-exposed private fields rather than making them public
